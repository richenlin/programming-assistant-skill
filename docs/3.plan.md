1. SOLUTION.md
   打开ChatGPT，告诉它：

我正在构建一个xxx（详细描述你的产品——功能、目标用户、关键特性，越详细越好）
前端使用 Vue.js，数据库使用 Postgre， 后端使用golang 1.22+。
请帮我生成完整的架构设计，包括：

- 文件与文件夹结构
- 每个部分的作用
- 状态存储位置
- 服务之间如何连接
  请将整个文档用markdown格式输出。

把ChatGPT的输出保存为SOLUTION.md，放到一个空文件夹中，这个文件夹将作为你的项目根目录

------------------------

2. TASK.md
   跟ChatGPT 4o说：

基于SOLUTION.md的架构，写一个详细的构建分步计划。
每个任务都应该：

- 保持准确精简且可测试
- 有明确的开始与结束
- 专注于一个问题
- 任务要按实现顺序排列
  
  

我会把这些任务交给一个工程LLM，一次只执行一个任务，能够测试每一个任务单元

将输出内容保存为TASK.md，放到项目文件夹中

------------------------

3. 回到Cursor，告诉它：

你是一名资深的软件工程师，正在开发这个软件项目。
  你已经拿到了SOLUTION.md和TASK.md。

**开发要求：**

- 仔细阅读这两个文件，确保对要构建的内容没有任何理解偏差

- 按TASK.md中的顺序，一次只完成一个任务

- 每完成一个任务就进行测试。如果测试通过，再继续下一个任务

- 不要在代码和文档中使用emoji；减少代码注释，非必要不编写注释；减少文档数量，仅保留README、SOLUTION、TASK、DEPLOYMENT等主要文档
  **开发守则：**

- 严格用最少的代码完成当前任务

- 不进行大规模改动

- 不做无关编辑，专注于你正在开发的任务

- 代码必须精确、模块化、可测试

- 不破坏现有功能
